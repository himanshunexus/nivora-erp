DEPLOYMENT METHODS AVAILABLE
================================================================================

Method 1: render.yaml (RECOMMENDED - Fastest)
  1. Go to https://render.com/dashboard
  2. Click "New +" → "Blueprint"
  3. Paste GitHub repo URL
  4. Authorize GitHub
  5. Configure DJANGO_SECRET_KEY and DJANGO_ALLOWED_HOSTS
  6. Click "Deploy"
  Time: ~5-10 minutes

Method 2: Manual Configuration (More Control)
  1. Create Web Service manually
  2. Create PostgreSQL database
  3. Create Redis cache
  4. Set environment variables
  5. Deploy
  Time: ~10-15 minutes

================================================================================
QUICK START - 5 MINUTES
================================================================================

1. Initialize Git (if not already done):
   $ git init
   $ git add .
   $ git commit -m "chore: prepare for Render deployment"
   $ git remote add origin https://github.com/USERNAME/nivora-erp.git
   $ git push -u origin main

2. Deploy on Render:
   a) Go to https://render.com
   b) Click "New +" → "Blueprint"
   c) Paste: https://github.com/USERNAME/nivora-erp.git
   d) Generate SECRET_KEY and set in Render
   e) Click "Deploy"

3. Wait for Deployment:
   - Build: ~2-3 minutes
   - Migrations: ~1 minute
   - Start: ~2 minutes
   Total: ~5-10 minutes

4. Access Application:
   https://<your-app>.onrender.com

================================================================================
TROUBLESHOOTING
================================================================================

Issue: "DJANGO_SECRET_KEY is required in production"
Fix: Set DJANGO_SECRET_KEY environment variable in Render

Issue: "Static files not found (404)"
Fix: Check build logs, ensure collectstatic runs, verify STATIC_ROOT

Issue: "Database connection error"
Fix: Verify DATABASE_URL is set, ensure PostgreSQL service is running

Issue: "500 Internal Server Error"
Fix: Check Render logs for detailed error message

Issue: "Migrations not applied"
Fix: Check Procfile release command, verify database connection

For complete troubleshooting, see: RENDER_DEPLOYMENT_GUIDE.md
